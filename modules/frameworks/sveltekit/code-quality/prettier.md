# SvelteKit + prettier

> Extends `modules/code-quality/prettier.md` with SvelteKit-specific integration.
> Generic prettier conventions (installation, `.prettierrc`, CI integration) are NOT repeated here.

## Integration Setup

```bash
npm install --save-dev prettier prettier-plugin-svelte
```

**`.prettierrc` for SvelteKit:**
```json
{
  "useTabs": true,
  "tabWidth": 2,
  "semi": true,
  "singleQuote": true,
  "printWidth": 100,
  "trailingComma": "es5",
  "plugins": ["prettier-plugin-svelte"],
  "overrides": [
    {
      "files": "*.svelte",
      "options": { "parser": "svelte" }
    }
  ]
}
```

## Framework-Specific Patterns

### `.prettierignore` for SvelteKit

SvelteKit generates `.svelte-kit/` on every `vite dev` start and produces a `build/` directory:

```
.svelte-kit/
build/
node_modules/
```

### Route file formatting

SvelteKit's file-based routing produces many small files. Prettier ensures consistent formatting across `+page.svelte`, `+page.server.ts`, `+layout.svelte`, and `+error.svelte`:

```bash
prettier --write "src/**/*.{svelte,ts,js,css,json}"
```

### Server vs. component file consistency

Prettier formats both server-side TypeScript (`+page.server.ts`) and client-side Svelte (`+page.svelte`) with the same base config. The `overrides` block applies the Svelte parser only to `.svelte` files, so server files use the standard TypeScript formatter.

### Svelte 5 runes formatting

Ensure `prettier-plugin-svelte >= 3.2.0` for Svelte 5 rune support. Earlier versions mangle rune expressions in `<script>` blocks.

## Additional Dos

- Add `.svelte-kit/` to `.prettierignore` — formatting generated files wastes time and produces noisy diffs.
- Run Prettier on both `.svelte` and `.ts` files in pre-commit hooks to cover all SvelteKit file types.

## Additional Don'ts

- Don't format `src/app.d.ts` — it's generated by SvelteKit and should not be manually edited or formatted.
- Don't use `prettier-plugin-tailwindcss` without testing — it may conflict with `prettier-plugin-svelte`; check plugin ordering if both are used.
