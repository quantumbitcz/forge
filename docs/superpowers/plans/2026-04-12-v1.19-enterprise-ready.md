# v1.19 Enterprise Ready — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make forge enterprise-ready with OpenTelemetry observability, OWASP security compliance, data classification, event-driven automations, async execution, and A2A protocol for cross-repo communication.

**Architecture:** Six features. Observability (Feature 11) and data classification (Feature 12) are foundational — they provide the telemetry and security infrastructure that other features reference. OWASP compliance (Feature 16) builds on both. Automations (Feature 13), async execution (Feature 14), and A2A (Feature 15) are independent. Implementation order follows dependencies.

**Tech Stack:** Markdown (contracts, agents), Bash (scripts), JSON (state schema, category registry), BATS (contract tests)

**Implementation Order:** Feature 11 (observability) → 12 (data classification) → 16 (OWASP) → 13 (automations) → 14 (async) → 15 (A2A) → Cross-cutting

In the orchestrator's stage transition logic, add:

```markdown
### §0.4a Telemetry Initialization (v1.19+)

If `observability.enabled`:
1. Initialize `state.json.telemetry` with empty spans and metrics
2. Create root span: `{ "name": "pipeline-run", "type": "pipeline", "start": now() }`
3. On each stage transition: create stage span, close previous stage span
4. On each agent dispatch: create agent span with model and token data
5. On pipeline completion: close root span, compute final metrics
6. If `observability.export == "otel"`: call `shared/forge-otel-export.sh`
```

- [ ] **Step 2: Add security enforcement to PREFLIGHT**

```markdown
### §0.4b Security Enforcement (v1.19+)

If `security.input_sanitization`:
1. Sanitize requirement text: strip HTML tags, script tags, markdown injection patterns
2. Log sanitized characters count in stage notes

If `security.convention_signatures`:
1. Compute SHA256 of each convention file loaded
2. Compare against stored signatures in `forge-config.md`
3. Mismatch: log WARNING "Convention file {path} signature mismatch — may have been tampered"

If `security.tool_call_budget`:
1. Store budget map: `agent_id → max_calls` from config
2. On each agent dispatch: pass budget to orchestrator tracking
3. If agent exceeds budget: force-stop agent, log WARNING, proceed with partial results
```
