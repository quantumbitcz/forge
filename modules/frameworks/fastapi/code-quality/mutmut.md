# FastAPI + mutmut

> Extends `modules/code-quality/mutmut.md` with FastAPI-specific integration.
> Generic mutmut conventions (mutation categories, CI caching, kill rate thresholds) are NOT repeated here.

## Integration Setup

Configure mutmut to use async-capable pytest with a short timeout per mutant:

```toml
[tool.mutmut]
paths_to_mutate = "app/"
tests_dir = "tests/unit/"
runner = "python -m pytest -x -q --timeout=10 --no-header -rN"
backup = false
```

**`pyproject.toml` pytest settings (required for runner async mode):**
```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
```

## Framework-Specific Patterns

### Async Test Patterns

mutmut's runner invokes pytest synchronously, but individual tests can be async. `asyncio_mode = "auto"` enables this without per-test decoration:

```python
# tests/unit/test_user_service.py
async def test_create_user_sets_hashed_password(
    mock_db: AsyncMock,
) -> None:
    service = UserService(db=mock_db)
    user = await service.create(email="user@example.com", password="plain")
    assert user.hashed_password != "plain"
    assert len(user.hashed_password) > 20
```

Use `AsyncMock` for dependency injection in unit tests — avoids database connections during mutant runs:

```python
from unittest.mock import AsyncMock
import pytest

@pytest.fixture
def mock_db() -> AsyncMock:
    return AsyncMock(spec=AsyncSession)
```

### Scoping to Domain and Service Layers

FastAPI routers contain framework boilerplate (dependency injection wiring, HTTP status codes) that produces noisy surviving mutants. Target mutations at service and domain logic:

```toml
[tool.mutmut]
# Target: service layer, domain models, utility functions
paths_to_mutate = "app/services/ app/domain/ app/utils/"
# Skip: routers (HTTP layer), schemas (Pydantic validation), dependencies (DI wiring)
tests_dir = "tests/unit/"
runner = "python -m pytest tests/unit/ -x -q --timeout=10"
backup = false
```

### Excluding Pydantic Validators

Pydantic v2 field validators and `model_validator` methods contain generated bytecode that mutmut cannot meaningfully test with unit tests alone. Scope `paths_to_mutate` to exclude schema files or use a wrapper:

```toml
[tool.mutmut]
# Mutate service logic but not Pydantic schema definitions
paths_to_mutate = "app/services/ app/repositories/ app/domain/"
```

If schemas contain custom validation logic worth testing, add targeted unit tests for the validators directly before enabling mutation on those files.

### httpx.AsyncClient in Unit Tests

Avoid using `httpx.AsyncClient` with the full FastAPI app in mutmut's unit test runner — it starts the app, resolves all dependencies, and multiplies per-mutant time significantly. Use direct service instantiation with mocked dependencies instead:

```python
# Preferred for mutmut unit tests — direct service call, no HTTP overhead
async def test_calculate_discount_applied_to_cart(
    mock_product_repo: AsyncMock,
) -> None:
    service = CartService(product_repo=mock_product_repo)
    total = await service.calculate_total(cart_id=1, coupon="SAVE10")
    assert total == pytest.approx(90.0)
```

## Additional Dos

- Set `asyncio_mode = "auto"` in `[tool.pytest.ini_options]` — mutmut's runner inherits this and async unit tests work without additional configuration.
- Use `AsyncMock` for database sessions and external service clients — prevents database setup overhead on every mutant execution.
- Scope `paths_to_mutate` to `app/services/` and `app/domain/` — router files and Pydantic schemas produce low-signal surviving mutants.

## Additional Don'ts

- Don't include integration tests using `httpx.AsyncClient` with the full ASGI app in the mutmut runner — app startup per mutant makes the run prohibitively slow.
- Don't mutate `app/routers/` without scoped unit tests for each route's business logic — most surviving mutants in router files reflect missing logic tests, not framework issues.
- Don't run mutmut without `--timeout=10` in the runner — async tests that deadlock (e.g., awaiting on a mock that never resolves) block indefinitely without a timeout.
