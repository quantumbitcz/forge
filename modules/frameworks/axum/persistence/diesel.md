# Axum + Diesel

> Axum-specific patterns for Diesel ORM. Extends generic Axum conventions.
> Generic Axum patterns (routing, error handling, extractors) are NOT repeated here.

## Integration Setup

`Cargo.toml`:
```toml
[dependencies]
diesel = { version = "2.2", features = ["postgres", "uuid", "chrono", "r2d2"] }
diesel-async = { version = "0.5", features = ["postgres", "deadpool"] }
deadpool-diesel = { version = "0.5", features = ["postgres"] }
dotenvy = "0.15"
```

## Connection Pool (deadpool-diesel, recommended for async)

```rust
use deadpool_diesel::postgres::{Manager, Pool};

pub type DbPool = Pool;

pub fn create_pool(database_url: &str) -> DbPool {
    let manager = Manager::new(database_url, deadpool_diesel::Runtime::Tokio1);
    Pool::builder(manager)
        .max_size(16)
        .build()
        .expect("Failed to create database pool")
}
```

## State Integration

```rust
#[derive(Clone)]
pub struct AppState {
    pub db: DbPool,
}

#[tokio::main]
async fn main() {
    let pool = create_pool(&std::env::var("DATABASE_URL").unwrap());
    let state = AppState { db: pool };

    let app = Router::new()
        .route("/users", get(list_users))
        .with_state(state);
}
```

## Handler Pattern with spawn_blocking

Diesel queries are synchronous — wrap them with `spawn_blocking` via deadpool-diesel's `interact`:

```rust
async fn list_users(State(state): State<AppState>) -> Result<Json<Vec<User>>, AppError> {
    let conn = state.db.get().await.map_err(|e| AppError::Pool(e.to_string()))?;

    let users = conn
        .interact(|conn| {
            use crate::schema::users::dsl::*;
            users.load::<User>(conn)
        })
        .await
        .map_err(|e| AppError::Interact(e.to_string()))?
        .map_err(|e| AppError::Database(e.to_string()))?;

    Ok(Json(users))
}
```

## diesel-async (alternative for fully async Diesel)

```toml
[dependencies]
diesel-async = { version = "0.5", features = ["postgres", "deadpool"] }
```

```rust
use diesel_async::{AsyncPgConnection, RunQueryDsl, AsyncConnection};

async fn find_user(pool: &AsyncPool, id: Uuid) -> QueryResult<User> {
    let mut conn = pool.get().await?;
    users::table.find(id).get_result(&mut conn).await
}
```

## Additional Dos/Don'ts

- DO use `deadpool-diesel` or `diesel-async` — never block the Tokio runtime with raw r2d2
- DO check `interact` errors separately from Diesel errors (two distinct error types)
- DO keep schema.rs auto-generated by `diesel print-schema` — do not hand-edit it
- DON'T use `.unwrap()` on pool `get()` — propagate errors through `AppError`
- DON'T share a `PgConnection` across `.await` points — connections are `!Send` in r2d2
