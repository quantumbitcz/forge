# S3 Storage with Vapor + Soto

## Integration Setup

```swift
// Package.swift — Soto is the community AWS SDK for Swift
.package(url: "https://github.com/soto-project/soto.git", from: "6.7.0"),
// targets:
.product(name: "SotoS3", package: "soto"),
```

```swift
// configure.swift
import SotoS3

extension Application {
    var aws: AWSClient {
        get { storage[AWSClientKey.self]! }
        set { storage[AWSClientKey.self] = newValue }
    }
    var s3: S3 { S3(client: aws, region: .useast1) }
}

// In configure()
app.aws = AWSClient(
    credentialProvider: .environment,      // AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
    httpClientProvider: .createNewWithEventLoopGroup(app.eventLoopGroup)
)
app.lifecycle.use(AWSLifecycle(client: app.aws))
```

## Framework-Specific Patterns

### Upload
```swift
func upload(req: Request) async throws -> Response {
    let data = try req.body.collect(max: 10 * 1024 * 1024)  // 10 MB limit
    let key  = "\(UUID())-\(req.parameters.get("filename") ?? "upload")"

    let putRequest = S3.PutObjectRequest(
        body:          .init(byteBuffer: data ?? .init()),
        bucket:        bucket,
        contentType:   req.headers.contentType?.description,
        key:           key
    )
    _ = try await req.application.s3.putObject(putRequest)
    return Response(status: .created, body: .init(string: key))
}
```

### Presigned URL (client-direct upload)
```swift
let presignedURL = try await req.application.s3.signURL(
    url: URL(string: "https://\(bucket).s3.amazonaws.com/\(key)")!,
    httpMethod: .PUT,
    expires: .minutes(15)
)
```

### Multipart Upload
```swift
let upload = try await s3.createMultipartUpload(
    S3.CreateMultipartUploadRequest(bucket: bucket, key: key)
)
// Upload parts in parallel, then complete
try await s3.completeMultipartUpload(
    S3.CompleteMultipartUploadRequest(
        bucket: bucket, key: key,
        multipartUpload: .init(parts: completedParts),
        uploadId: upload.uploadId!
    )
)
```

## Additional Dos/Don'ts

- DO use presigned URLs for client-direct uploads to avoid routing large payloads through Vapor
- DO shut down `AWSClient` via lifecycle hook — `app.lifecycle.use(AWSLifecycle(client: app.aws))`
- DO set `contentType` on `PutObjectRequest` for correct browser rendering of stored files
- DO enforce file size limits before streaming to S3 (`req.body.collect(max:)`)
- DON'T hardcode AWS credentials — use environment variables or IAM instance roles
- DON'T store large file buffers in memory; use streaming upload for files > 5 MB
- DON'T expose bucket names or object keys in public responses without authorization checks
