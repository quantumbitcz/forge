# forge 3.2.0 — Phase 03 Prompt Injection Hardening

Release date: 2026-04-19.
Closes audit finding **W3** (unfiltered external data concatenated into agent prompts).

## Highlights

- **Four-tier trust model** for every external data source (Silent / Logged / Confirmed / Blocked).
- **All external data wrapped** in `<untrusted>` XML envelopes by `hooks/_py/mcp_response_filter.py` before reaching any agent.
- **Standard Untrusted Data Policy block** SHA-pinned and injected into all 42 agents.
- **Regex pattern library** (`shared/prompt-injection-patterns.json`) — 40 seed patterns across 7 categories: OVERRIDE, ROLE_HIJACK, SYSTEM_SPOOF, TOOL_COERCION, EXFIL, CREDENTIAL_SHAPED, PROMPT_LEAK.
- **Forensic event log** at `.forge/security/injection-events.jsonl` (one record per filter invocation; non-rotating).
- **Seven new scoring categories**: `SEC-INJECTION-OVERRIDE`, `SEC-INJECTION-EXFIL`, `SEC-INJECTION-TOOL-MISUSE`, `SEC-INJECTION-BLOCKED`, `SEC-INJECTION-TRUNCATED`, `SEC-INJECTION-DISABLED`, `SEC-INJECTION-HISTORICAL` — all routed to `fg-411-security-reviewer`.
- **Confirmed-tier + Bash gate**: orchestrator MUST call `AskUserQuestion` before dispatching any `Bash`-capable agent that will receive Confirmed-tier data — **even in `autonomous: true` mode**. Background/CI fallback: `.forge/alerts.json` + pause.
- **New error type** `INJECTION_BLOCKED` (CRITICAL, not recoverable by retry — source must be fixed).
- **PREFLIGHT halts** when `security.untrusted_envelope.enabled` or `security.injection_detection.enabled` is set to `false`.
- **One-time PREFLIGHT retro-scan** of pre-3.2.0 `.forge/wiki/` and `.forge/explore-cache.json` emits `SEC-INJECTION-HISTORICAL` INFO findings (informational; does not halt).
- **Token overhead**: ~150 tokens per dispatched agent (≈ 2,700 tokens per 18-agent run; 0.14% of typical 2 M run budget).

## Breaking changes

1. Every `agents/*.md` grows by a 120-word canonical header. Tools that parse "H1 immediately followed by `## 1.`" must now skip the policy block.
2. MCP tool outputs are no longer concatenated directly into prompts. Subscribers must use `hooks/_py/mcp_response_filter.py`.
3. Third-party agent extensions must carry the standard header or fail the `tests/structural/untrusted-header-present.bats` structural check.
4. `security.untrusted_envelope.enabled: false` is a configuration error — remove any such override before upgrading.
5. Autonomous mode pauses when Confirmed-tier (T-C) data feeds a `Bash`-capable agent.

## Migration

No feature flag. Re-run `/forge-init` on upgrade; config validation force-adds the `security.*` block at defaults. Pre-existing `.forge/` state is preserved; the historical retro-scan runs once at next PREFLIGHT.

## Success criteria (all met)

- Adversarial eval: 0 successful injections across 10 scenarios (`tests/evals/scenarios/injection-redteam/01-10`).
- All 42 agents carry the canonical header (SHA256-verified by `tools/verify-untrusted-header.sh`).
- Pattern library ships with 40 entries covering all 7 categories.
- Baseline `SEC-INJECTION-*` finding count on a clean run is 0.
- Per-run token overhead under the 10,000 budget (measured: ~2,646).
