# Axum + AWS S3

> Axum-specific S3 patterns using `aws-sdk-s3`.
> Extends generic Axum conventions.

## Integration Setup

```toml
# Cargo.toml
[dependencies]
axum = "0.8"
aws-config = { version = "1.5", features = ["behavior-version-latest"] }
aws-sdk-s3 = "1.38"
tokio = { version = "1", features = ["full"] }
uuid = { version = "1", features = ["v4"] }
```

## S3 Client Setup

```rust
use aws_config::BehaviorVersion;
use aws_sdk_s3::Client;

pub async fn create_s3_client() -> Client {
    let config = aws_config::load_defaults(BehaviorVersion::latest()).await;
    Client::new(&config)
}
```

Credentials are resolved from environment (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`), EC2/ECS instance role, or Web Identity Token — never hardcode them.

## Presigned Upload URL

```rust
use aws_sdk_s3::presigning::PresigningConfig;
use std::time::Duration;

async fn presign_upload(
    State(state): State<AppState>,
    Query(params): Query<PresignQuery>,
) -> Result<Json<PresignResponse>, AppError> {
    let key = format!("uploads/{}/{}", uuid::Uuid::new_v4(), params.filename);

    let presigned = state.s3
        .put_object()
        .bucket(&state.bucket)
        .key(&key)
        .presigned(
            PresigningConfig::expires_in(Duration::from_secs(900))?,
        )
        .await?;

    Ok(Json(PresignResponse {
        upload_url: presigned.uri().to_string(),
        key,
        expires_in: 900,
    }))
}
```

## Multipart Upload Handler

```rust
use axum::extract::Multipart;

async fn upload_file(
    State(state): State<AppState>,
    mut multipart: Multipart,
) -> Result<Json<UploadResponse>, AppError> {
    while let Some(field) = multipart.next_field().await? {
        let content_type = field.content_type()
            .unwrap_or("application/octet-stream")
            .to_string();
        let data = field.bytes().await?;

        // Validate size
        if data.len() > 50 * 1024 * 1024 {
            return Err(AppError::BadRequest("file too large".into()));
        }

        let key = format!("uploads/{}", uuid::Uuid::new_v4());
        state.s3
            .put_object()
            .bucket(&state.bucket)
            .key(&key)
            .content_type(content_type)
            .body(data.into())
            .send()
            .await?;

        return Ok(Json(UploadResponse { key }));
    }
    Err(AppError::BadRequest("no file field".into()))
}
```


## Additional Dos/Don'ts

- DO validate file size before calling `put_object` — reject oversized payloads early
- DO use presigned URLs for large file uploads — avoids loading the entire file into Axum handler memory
- DO generate a UUID prefix for every key to prevent filename collisions
- DO inject the bucket name from environment into `AppState` — never hardcode it
- DON'T use `get_object` to proxy file downloads through your API server — generate presigned `get_object` URLs instead
- DON'T return the internal S3 key to untrusted clients without validation — use opaque IDs mapped to keys in your database
