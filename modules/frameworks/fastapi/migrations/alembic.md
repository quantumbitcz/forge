## Framework-Specific Patterns

### env.py Async Support

```python
# alembic/env.py — key wiring (alembic init -t async generates the boilerplate)
from app.models import Base       # Import all models so autogenerate detects tables
from app.config import settings

config.set_main_option("sqlalchemy.url", settings.database_url)
target_metadata = Base.metadata

# Use async_engine_from_config + connection.run_sync for the migration runner
# (generated by `alembic init -t async`; fill in target_metadata and url above)
```

Import every model module in `env.py` (or a central `app/models/__init__.py`) — autogenerate only sees tables whose models are imported.

### Autogenerate from SQLAlchemy Models

```bash
# Generate a new migration
alembic revision --autogenerate -m "add status column to orders"

# Always review generated migrations before applying
alembic upgrade head
```

### Migration in Docker Entrypoint

```dockerfile
# Dockerfile
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
```

```bash
#!/bin/sh
# entrypoint.sh
set -e
alembic upgrade head
exec uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Running migrations at entrypoint (not import time) ensures K8s rolling deploys apply schema before new pods serve traffic.

### Downgrade Safety

```python
# In generated migration — always fill in downgrade()
def downgrade() -> None:
    op.drop_column("orders", "status")   # Never leave as `pass`
```

## Scaffolder Patterns

```yaml
patterns:
  alembic_ini: "alembic.ini"
  env_py: "alembic/env.py"
  versions_dir: "alembic/versions/"
  migration_file: "alembic/versions/{timestamp}_{slug}.py"
  entrypoint: "entrypoint.sh"
```

## Additional Dos/Don'ts

- DO import all model modules in `env.py` before `target_metadata = Base.metadata`
- DO review autogenerated migrations — Alembic misses some changes (e.g., column type precision, server defaults)
- DO implement `downgrade()` for every migration — empty downgrades break rollback pipelines
- DON'T set `sqlalchemy.url` directly in `alembic.ini` — read it from app settings in `env.py` to avoid credential leaks
- DON'T run `alembic upgrade head` inside the FastAPI lifespan — use the entrypoint script
