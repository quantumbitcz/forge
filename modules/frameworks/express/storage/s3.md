# Express + AWS S3

> Express-specific patterns for object storage via @aws-sdk/client-s3. Extends generic Express conventions.

## Integration Setup

```bash
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner multer multer-s3
npm install --save-dev @types/multer
```

```typescript
// src/lib/s3.ts
import { S3Client } from "@aws-sdk/client-s3";

export const s3 = new S3Client({
  region: process.env.AWS_REGION ?? "eu-central-1",
  endpoint: process.env.AWS_ENDPOINT,         // LocalStack override
  forcePathStyle: !!process.env.AWS_ENDPOINT, // required for LocalStack
});

export const S3_BUCKET = process.env.S3_BUCKET!;
```

## Framework-Specific Patterns

### multer-s3 direct upload

```typescript
// src/middleware/upload.ts
import multer from "multer";
import multerS3 from "multer-s3";
import { v4 as uuid } from "uuid";
import { s3, S3_BUCKET } from "../lib/s3";

export const uploadToS3 = multer({
  storage: multerS3({
    s3,
    bucket: S3_BUCKET,
    contentType: multerS3.AUTO_CONTENT_TYPE,
    key: (_req, file, cb) => cb(null, `uploads/${uuid()}/${file.originalname}`),
  }),
  limits: { fileSize: 10 * 1024 * 1024 }, // 10 MB
});
```

```typescript
// Route usage
router.post("/files", uploadToS3.single("file"), (req, res) => {
  const file = req.file as Express.MulterS3.File;
  res.status(201).json({ key: file.key });
});
```

### Presigned URL endpoint

```typescript
import { GetObjectCommand, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

router.get("/files/:key(*)/url", async (req, res, next) => {
  try {
    const url = await getSignedUrl(
      s3,
      new GetObjectCommand({ Bucket: S3_BUCKET, Key: req.params.key }),
      { expiresIn: 900 }
    );
    res.json({ url });
  } catch (err) {
    next(err);
  }
});
```

### Stream upload (large files without buffering)

```typescript
import { PutObjectCommand } from "@aws-sdk/client-s3";

router.put("/files/:key(*)", async (req, res, next) => {
  try {
    await s3.send(new PutObjectCommand({
      Bucket: S3_BUCKET,
      Key: req.params.key,
      Body: req,                              // pipe request stream directly
      ContentType: req.headers["content-type"],
      ContentLength: Number(req.headers["content-length"]),
    }));
    res.status(204).end();
  } catch (err) {
    next(err);
  }
});
```

## Scaffolder Patterns

```
src/
  lib/
    s3.ts                 # S3Client singleton + S3_BUCKET
  middleware/
    upload.ts             # multer-s3 configuration
  routes/
    storage.ts            # presigned URL + stream endpoints
```

## Dos

- Use `forcePathStyle: true` when `AWS_ENDPOINT` is set — required for LocalStack and MinIO compatibility
- Validate `content-type` and enforce `limits.fileSize` in multer before the file reaches S3
- Return only the object key to the client; generate presigned URLs on demand rather than storing them
- Use `getSignedUrl` with short TTL (≤15 minutes) for download links served to end users

## Don'ts

- Don't expose raw S3 URLs — they are public if the bucket ACL is not private
- Don't buffer multipart uploads in memory — stream directly using `multer-s3` or pipe `req` to `PutObjectCommand`
- Don't catch all errors as 500 — inspect `err.$metadata.httpStatusCode` for 403/404 from S3
- Don't commit `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` — use IAM roles or environment injection
