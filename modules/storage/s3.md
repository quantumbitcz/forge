# AWS S3 — Object Storage Best Practices

## Overview

Amazon S3 is the industry-standard object storage service, offering 11 nines of durability and
virtually unlimited capacity. Use S3 for user-uploaded files, static assets, backups, data lake
storage, and event-driven processing pipelines. S3 is not a file system — object keys are flat
(folders are a UI convention), and there are no atomic multi-object transactions. Design access
patterns around individual object operations and pre-signed URLs for client-side uploads/downloads.

## Architecture Patterns

### Pre-Signed URLs (Client Direct Upload/Download)
```python
import boto3
s3 = boto3.client("s3")

# Pre-signed PUT — client uploads directly, never through your server
put_url = s3.generate_presigned_url(
    "put_object",
    Params={"Bucket": "my-uploads", "Key": f"users/{user_id}/avatar.jpg",
            "ContentType": "image/jpeg", "ContentLength": max_size_bytes},
    ExpiresIn=300   # 5 minutes
)

# Pre-signed GET — expiring download link for private objects
get_url = s3.generate_presigned_url(
    "get_object",
    Params={"Bucket": "my-private", "Key": object_key,
            "ResponseContentDisposition": "attachment; filename=report.pdf"},
    ExpiresIn=3600
)
```
Pre-signed URLs offload bandwidth to S3 and remove your server from the data path entirely.

### Multipart Upload (Files > 100 MB)
```python
# Initiate
mpu = s3.create_multipart_upload(Bucket="my-bucket", Key="large-file.zip")
upload_id = mpu["UploadId"]

# Upload parts (each part >= 5 MB except the last)
parts = []
for i, chunk in enumerate(read_chunks(file_path, chunk_size=50_000_000), start=1):
    part = s3.upload_part(Bucket="my-bucket", Key="large-file.zip",
                           UploadId=upload_id, PartNumber=i, Body=chunk)
    parts.append({"PartNumber": i, "ETag": part["ETag"]})

# Complete
s3.complete_multipart_upload(
    Bucket="my-bucket", Key="large-file.zip",
    UploadId=upload_id, MultipartUpload={"Parts": parts}
)
```
Abort incomplete multipart uploads automatically via lifecycle policy to avoid storage costs:
```json
{ "Rules": [{ "ID": "abort-incomplete-mpu", "Status": "Enabled",
              "AbortIncompleteMultipartUpload": { "DaysAfterInitiation": 7 } }] }
```

### Lifecycle Policies (Transition and Expiration)
```json
{
  "Rules": [{
    "ID": "logs-tiering",
    "Status": "Enabled",
    "Filter": { "Prefix": "logs/" },
    "Transitions": [
      { "Days": 30,  "StorageClass": "STANDARD_IA" },
      { "Days": 90,  "StorageClass": "GLACIER_IR" },
      { "Days": 365, "StorageClass": "DEEP_ARCHIVE" }
    ],
    "Expiration": { "Days": 2555 }   // 7 years
  }]
}
```

### Versioning and Server-Side Encryption
```python
# Enable versioning (required for MFA delete and cross-region replication)
s3.put_bucket_versioning(
    Bucket="my-bucket",
    VersioningConfiguration={"Status": "Enabled"}
)

# SSE-KMS (recommended for compliance workloads)
s3.put_object(
    Bucket="my-bucket",
    Key="sensitive/data.json",
    Body=json_data,
    ServerSideEncryption="aws:kms",
    SSEKMSKeyId="arn:aws:kms:us-east-1:123456:key/abc-def"
)
```

### Event Notifications (S3 → SQS/SNS/Lambda)
```python
s3.put_bucket_notification_configuration(
    Bucket="my-uploads",
    NotificationConfiguration={
        "QueueConfigurations": [{
            "QueueArn": "arn:aws:sqs:us-east-1:123456:image-processing",
            "Events": ["s3:ObjectCreated:*"],
            "Filter": { "Key": { "FilterRules": [
                { "Name": "prefix", "Value": "users/" },
                { "Name": "suffix", "Value": ".jpg" }
            ]}}
        }]
    }
)
```

### Access Points (Fine-Grained Access)
```python
# Create an access point scoped to a specific prefix for a microservice
s3control.create_access_point(
    AccountId="123456789",
    Name="image-processor-ap",
    Bucket="my-shared-bucket",
    VpcConfiguration={"VpcId": "vpc-abc123"},
    PublicAccessBlockConfiguration={"BlockPublicAcls": True, "BlockPublicPolicy": True,
                                    "IgnorePublicAcls": True, "RestrictPublicBuckets": True}
)
```

## Configuration

```python
# Recommended client config (boto3)
import boto3
from botocore.config import Config

s3 = boto3.client("s3", config=Config(
    retries={"max_attempts": 3, "mode": "adaptive"},
    max_pool_connections=50
))
```

**Bucket-level settings:**
- Block all public access unless serving a static website.
- Enable versioning on buckets containing critical data.
- Enable access logging to a separate bucket for audit trails.
- Enforce SSE via bucket policy: deny `s3:PutObject` without `aws:kms` condition.

## Performance

- Use `Transfer Acceleration` for clients geographically distant from the bucket region.
- Use S3 byte-range GETs to parallelize large downloads: `Range: bytes=0-52428799`.
- Key prefix design: avoid timestamp or sequential prefixes for high-throughput workloads (old
  guidance; S3 now auto-partitions, but mixed-prefix keys still perform better for > 3500 PUT/s).
- Use `S3 Select` to retrieve a subset of CSV/JSON/Parquet data without downloading the full object.

## Security

- Block all public access at the account level unless specifically required.
- Use IAM roles (not access keys) for EC2/Lambda/ECS access to S3.
- Enforce HTTPS: deny `s3:*` where `aws:SecureTransport = false` in bucket policy.
- Rotate pre-signed URLs with short expiration (5-60 minutes for uploads, 1-24 hours for downloads).
- Enable S3 Object Lock for compliance buckets (WORM — Write Once Read Many).
- Use S3 Access Analyzer to detect overly permissive bucket policies.

## Testing

```python
# Localstack for S3 integration tests (no AWS account needed)
import boto3

s3 = boto3.client("s3", endpoint_url="http://localhost:4566",
                  aws_access_key_id="test", aws_secret_access_key="test")
s3.create_bucket(Bucket="test-bucket")
s3.put_object(Bucket="test-bucket", Key="test.txt", Body=b"hello")
obj = s3.get_object(Bucket="test-bucket", Key="test.txt")
assert obj["Body"].read() == b"hello"
```

## Dos
- Use pre-signed URLs for client uploads/downloads — they remove your server from the data path.
- Enable versioning on critical data buckets for accidental-deletion recovery.
- Set lifecycle policies to automatically transition infrequently accessed objects to cheaper storage classes.
- Enforce SSE-KMS for regulated data (PII, financial, health records).
- Use IAM roles, not long-lived access keys, for service-to-S3 authentication.

## Don'ts
- Don't make S3 buckets public without deliberate intent and explicit review.
- Don't store credentials or secrets in S3 objects — use AWS Secrets Manager or Parameter Store.
- Don't use sequential numeric prefixes in high-throughput workloads — distribute key prefixes.
- Don't skip multipart upload for files > 100 MB — single-part upload has a 5 GB limit and no retry.
- Don't use the root AWS account credentials in any application code.
