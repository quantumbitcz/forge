# dart-format

## Overview

Dart's built-in formatter, shipped with the Dart SDK. Enforces the official Dart style guide: 2-space indentation, no trailing whitespace, consistent operator spacing. Available as `dart format` (Dart 2.14+, replaces deprecated `dartfmt`). Use `--set-exit-if-changed` in CI to fail on unformatted code without writing. Line length is the primary configurable option via `--line-length`; the standard is 80 characters. The formatter is deterministic and idempotent — running it twice produces the same output.

## Architecture Patterns

### Installation & Setup

```bash
# dart format ships with the Dart SDK — no separate install
dart --version   # confirms Dart SDK version

# Flutter projects use the same formatter
flutter --version

# Verify
dart format --help
```

No configuration file — `dart format` accepts options only via CLI flags. Project-level conventions should be documented in `analysis_options.yaml` comments or a `Makefile`.

**`analysis_options.yaml` (linting, not formatting — but complements formatting):**
```yaml
include: package:lints/recommended.yaml

analyzer:
  exclude:
    - "**/*.g.dart"       # generated files (build_runner)
    - "**/*.freezed.dart" # freezed generated files
    - "lib/generated/**"

linter:
  rules:
    - always_declare_return_types
    - prefer_const_constructors
    - prefer_final_locals
    - avoid_print
```

### Rule Categories

`dart format` is not rule-based — it applies a fixed style. Key behaviors:

| Behavior | Description |
|---|---|
| Indentation | 2 spaces, no tabs |
| Line length | Default 80 (configurable via `--line-length`) |
| Trailing commas | Trailing commas in function calls/params force vertical layout |
| Import sorting | NOT sorted by `dart format` — use `import_sorter` or `dart fix` |
| Cascade operators | `..` on new line for multi-step cascades |

**Trailing comma convention (important for Flutter):**
```dart
// Trailing comma → dart format keeps this vertical (readable in Flutter widget trees)
Widget build(BuildContext context) {
  return Column(
    children: [
      Text('Hello'),
      Text('World'),   // ← trailing comma
    ],
  );
}

// No trailing comma → dart format may collapse to one line
```

### Configuration Patterns

**`Makefile` targets (no config file):**
```makefile
.PHONY: format format-check

format:
	dart format .

format-check:
	dart format --set-exit-if-changed --output=none .

format-check-diff:
	dart format --set-exit-if-changed .
```

**For Flutter projects with generated files:**
```bash
# Format only non-generated source files
dart format --exclude "**/*.g.dart" --exclude "**/*.freezed.dart" lib/ test/
```

**Custom line length (e.g., for large monitors or wide terminals):**
```bash
dart format --line-length 100 .
```

**`pubspec.yaml` dev script entry (using `melos` for monorepos):**
```yaml
# melos.yaml
scripts:
  format:
    run: dart format .
    description: Format all Dart files

  format:check:
    run: dart format --set-exit-if-changed --output=none .
    description: Check formatting without writing
```

```yaml
# .github/workflows/quality.yml
- name: Check Dart formatting
```

**Flutter CI:**

```yaml

repos:
  - repo: local
    hooks:
      - id: dart-format
        name: dart format
        entry: dart format

        files: \.dart$

```

## Performance

- `dart format .` on a typical Flutter app (300 files) completes in 2-5 seconds.
- Use path scoping (`dart format lib/ test/`) to exclude generated code directories (`build/`, `.dart_tool/`).
- `--output=none` skips writing formatted content to stdout — significantly faster in `--check` mode for large projects.
- For monorepos managed with `melos`, run `dart format` scoped to changed packages: `melos exec -- dart format .`.

## Security

dart format has no security analysis capability. Key practices:

- `dart format` does not execute code — safe to run on untrusted source.
- The Dart SDK ships dart format — no additional supply chain risk from the formatter itself.
- Generated files (`.g.dart`, `.freezed.dart`) should be excluded from formatting — they are regenerated by `build_runner` and may contain layout that the tool expects to control.

## Testing

```bash
# Check all files without writing (CI mode)
dart format --set-exit-if-changed --output=none .

# Format all files in place
dart format .

# Format a specific file
dart format lib/src/my_widget.dart

# Format with custom line length
dart format --line-length 100 .

# Show what would change (writes formatted output to stdout, not file)
dart format --output=show lib/src/my_widget.dart

# Verify a single file would not change
dart format --set-exit-if-changed lib/src/my_widget.dart

# Format lib/ and test/ only (exclude generated files)
dart format lib/ test/
```

## Dos

- Use `--set-exit-if-changed` in CI — without it, `dart format` exits 0 even when files are unformatted.
- Use `--output=none` with `--set-exit-if-changed` in CI — it avoids writing changed files to stdout (faster, cleaner logs).
- Use trailing commas in Flutter widget constructors — they preserve vertical formatting for widget trees, which is critical for readability.
- Exclude generated files (`*.g.dart`, `*.freezed.dart`) from formatting — they are regenerated by `build_runner` and do not need manual formatting.
- Set a consistent `--line-length` across all project scripts, CI, and editor configs — inconsistent lengths cause formatting churn.
- Configure your editor to run `dart format` on save — prevents accumulation of formatting fixes in PRs.

## Don'ts

- Don't use `dartfmt` — it is deprecated and removed in Dart 2.14+. Use `dart format` exclusively.
- Don't run `dart format` on the entire repo root without scoping — it processes `build/` and `.dart_tool/` directories unless explicitly excluded.
- Don't skip `--set-exit-if-changed` in CI and rely on `dart format` output parsing — the flag is the correct way to detect unformatted files.
- Don't add `dart format` output to `.gitignore` — the formatter output is not a build artifact; formatted source files belong in version control.
- Don't mix line length settings between `dart format --line-length` and `analysis_options.yaml` `max_line_length` — they are independent and mismatches cause lint warnings on formatter-correct lines.
- Don't configure Flutter widget builder closures without trailing commas — `dart format` will collapse them to single lines, breaking Flutter's widget tree visualization in IDE debuggers.
