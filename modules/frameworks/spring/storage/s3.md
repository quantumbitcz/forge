# Spring + AWS S3

> Spring-specific patterns for object storage via AWS SDK v2 S3Client. Extends generic Spring conventions.

## Integration Setup

```kotlin
// build.gradle.kts
implementation(platform("software.amazon.awssdk:bom:2.25.0"))
implementation("software.amazon.awssdk:s3")
implementation("software.amazon.awssdk:s3-transfer-manager")  // multipart
```

```yaml
# application.yml
aws:
  s3:
    region: ${AWS_REGION:eu-central-1}
    bucket: ${S3_BUCKET}
    endpoint: ${AWS_ENDPOINT:}           # override for LocalStack in tests
    presigned-url-ttl-minutes: 15
```

```kotlin
// S3Config.kt
@Configuration
class S3Config(@Value("\${aws.s3.region}") private val region: String,
               @Value("\${aws.s3.endpoint:}") private val endpoint: String) {

    @Bean
    fun s3Client(): S3Client = S3Client.builder()
        .region(Region.of(region))
        .apply { if (endpoint.isNotBlank()) endpointOverride(URI.create(endpoint)) }
        .credentialsProvider(DefaultCredentialsProvider.create())
        .build()

    @Bean
    fun s3Presigner(): S3Presigner = S3Presigner.builder()
        .region(Region.of(region))
        .apply { if (endpoint.isNotBlank()) endpointOverride(URI.create(endpoint)) }
        .build()

    @Bean
    fun s3TransferManager(s3Client: S3Client): S3TransferManager =
        S3TransferManager.builder().s3Client(
            S3AsyncClient.builder()
                .region(Region.of(region))
                .multipartEnabled(true)
                .build()
        ).build()
}
```

## Framework-Specific Patterns

```kotlin
@Service
class StorageService(
    private val s3: S3Client,
    private val presigner: S3Presigner,
    @Value("\${aws.s3.bucket}") private val bucket: String,
    @Value("\${aws.s3.presigned-url-ttl-minutes}") private val ttlMinutes: Long,
) {
    fun upload(key: String, bytes: ByteArray, contentType: String) {
        s3.putObject(
            PutObjectRequest.builder().bucket(bucket).key(key)
                .contentType(contentType).contentLength(bytes.size.toLong()).build(),
            RequestBody.fromBytes(bytes)
        )
    }

    fun presignedGetUrl(key: String): URI = presigner.presignGetObject {
        it.signatureDuration(Duration.ofMinutes(ttlMinutes))
        it.getObjectRequest { r -> r.bucket(bucket).key(key) }
    }.url().toURI()

    fun presignedPutUrl(key: String, contentType: String): URI = presigner.presignPutObject {
        it.signatureDuration(Duration.ofMinutes(ttlMinutes))
        it.putObjectRequest { r -> r.bucket(bucket).key(key).contentType(contentType) }
    }.url().toURI()

    fun delete(key: String) {
        s3.deleteObject(DeleteObjectRequest.builder().bucket(bucket).key(key).build())
    }
}
```

S3 health indicator (Actuator):

```kotlin
@Component("s3")
class S3HealthIndicator(private val s3: S3Client,
                        @Value("\${aws.s3.bucket}") private val bucket: String) : HealthIndicator {
    override fun health(): Health = runCatching {
        s3.headBucket(HeadBucketRequest.builder().bucket(bucket).build())
        Health.up().build()
    }.getOrElse { Health.down(it).build() }
}
```


## Dos

- Use presigned URLs for client-side direct upload/download — never proxy large payloads through the app
- Use `S3TransferManager` with multipart for files > 8 MB — it handles part splitting automatically
- Configure the S3 endpoint override via properties for LocalStack compatibility in integration tests
- Set `contentType` and `contentLength` explicitly on `PutObjectRequest` to avoid chunked transfer issues

## Don'ts

- Don't hardcode bucket names or region — always externalize via `@Value` or `@ConfigurationProperties`
- Don't return raw S3 URLs to clients in production — presigned URLs expire and enforce access control
- Don't store credentials in application config — use IAM roles (EC2/ECS/EKS) or `DefaultCredentialsProvider`
- Don't catch `SdkException` broadly; distinguish `NoSuchKeyException` (404) from transient network errors
