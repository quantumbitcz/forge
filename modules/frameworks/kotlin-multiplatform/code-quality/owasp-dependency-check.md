# Kotlin Multiplatform + OWASP Dependency-Check

> Extends `modules/code-quality/owasp-dependency-check.md` with Kotlin Multiplatform-specific integration.
> Generic OWASP Dependency-Check conventions are NOT repeated here.

## Integration Setup

```kotlin
// root build.gradle.kts (or shared/build.gradle.kts)
plugins {
    id("org.owasp.dependencycheck") version "9.0.9"
}

dependencyCheck {
    failBuildOnCVSS = 7.0f
    format = "ALL"
    suppressionFile = "config/owasp-suppressions.xml"
    nvd {
        apiKey = System.getenv("NVD_API_KEY") ?: ""
        delay = 4000
    }
    analyzers {
        assemblyEnabled = false
        nuspecEnabled = false
        // Enable CocoaPods analyzer for iOS dependency scanning
        cocoapodsEnabled = true
        // Enable NPM analyzer if the project has a jsMain target
        nodeEnabled = true
        nodeAuditEnabled = true
    }
}
```

## Framework-Specific Patterns

### Multi-Platform Dependency Surface

KMP projects aggregate dependencies across multiple platforms. Each platform has its own vulnerability surface:

| Platform Target | Dependency Manager | Scanner |
|---|---|---|
| `commonMain` | Gradle (Maven Central) | OWASP Dependency-Check (JVM) |
| `androidMain` | Gradle (Maven Central + Google) | OWASP Dependency-Check (JVM) |
| `iosMain` | CocoaPods / SPM | OWASP + `cocoapodsEnabled` |
| `jsMain` / `wasmMain` | npm | `nodeEnabled` + `nodeAuditEnabled` |

### Scanning iOS Dependencies (CocoaPods)

Enable CocoaPods analysis to scan `Podfile.lock`:

```kotlin
dependencyCheck {
    analyzers {
        cocoapodsEnabled = true
        // Point to the iOS app directory containing Podfile.lock
        // (OWASP auto-discovers Podfile.lock files in the scan path)
    }
    // Include iOS directory in the scan
    scanPath = "${rootDir}/iosApp"
}
```

### Scanning JS Dependencies (jsMain)

If the project has a `jsMain` target with npm dependencies (via Kotlin/JS Yarn/npm integration), include `node_modules` in the scan:

```kotlin
dependencyCheck {
    analyzers {
        nodeEnabled = true
        nodeAuditEnabled = true
    }
    // node_modules generated by Kotlin/JS in build directory
    scanPath = "${rootDir}/build/js"
}
```

### Separate CI Jobs per Platform

Run platform-specific dependency scans in separate CI jobs for clearer failure attribution:

```yaml
# .github/workflows/security.yml
jobs:
  dep-check-jvm:
    name: Dependency Check (JVM/Android)
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: "17", distribution: temurin }
      - name: Cache NVD database
        uses: actions/cache@v4
        with:
          path: ~/.gradle/dependency-check-data
          key: nvd-${{ github.run_id }}
          restore-keys: nvd-
      - run: ./gradlew dependencyCheckAnalyze
        env:
          NVD_API_KEY: ${{ secrets.NVD_API_KEY }}

  dep-check-js:
    name: Dependency Check (JS/npm)
    if: "contains(github.event.head_commit.message, 'js')"
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: "20" }
      - run: npm audit --audit-level=high
        working-directory: build/js/packages
```

### Common KMP Suppressions

Kotlin stdlib and Ktor transitive dependencies occasionally match unrelated CVEs:

```xml
<!-- config/owasp-suppressions.xml -->
<suppressions>
  <suppress>
    <notes>False positive ‚Äî CVE matches unrelated Apache project, not Kotlin stdlib.</notes>
    <packageUrl regex="true">^pkg:maven/org\.jetbrains\.kotlin:kotlin-stdlib.*$</packageUrl>
    <cve>CVE-XXXX-XXXXX</cve>
  </suppress>
</suppressions>
```

## Additional Dos

- Enable `cocoapodsEnabled` if the iOS app uses CocoaPods ‚Äî SPM dependencies are currently not scanned by OWASP Dependency-Check.
- Enable `nodeEnabled` for `jsMain` targets ‚Äî JS npm dependencies have a higher CVE frequency than JVM dependencies.
- Scan the `shared` module and each platform app module separately for clear per-platform reports.
- Cache `~/.gradle/dependency-check-data` across CI runs ‚Äî the NVD database is large and slow to download.

## Additional Don'ts

- Don't run a single `dependencyCheckAnalyze` that tries to scan all platform outputs in one pass ‚Äî it produces cross-platform false positives.
- Don't skip CocoaPods scanning if `Podfile.lock` is committed ‚Äî iOS pods have their own CVE history independent of JVM dependencies.
- Don't suppress multi-platform framework CVEs (Ktor, kotlinx-coroutines) without verifying the CVE actually applies to the specific version in use.
- Don't include SPM (Swift Package Manager) packages in the OWASP scan scope ‚Äî the analyzer does not support Swift packages and will report errors.
